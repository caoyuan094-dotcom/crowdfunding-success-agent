# 案例知识库整理框架

该文件用于沉淀公众号文章、过往客户项目、Kickstarter / Indiegogo 案例和团队经验。

原则：

- 不直接复制第三方文章。
- 只提炼可复用的评估项、风险项、话术、项目套路和案例标签。
- 未读正文或未验证来源，不得写成事实结论。

## 1. 翻车实录类

用于提炼失败原因，并转成红色风险检查项。

记录格式：

| 来源 | 项目/品类 | 翻车原因 | 对应风险项 | 智能体检查规则 |
|---|---|---|---|---|
|  |  |  |  |  |

常见标签：

- 无样机
- 认证缺失
- 毛利不足
- 交付延期
- 页面转化差
- 流量冷启动
- 价格过高
- 疑似侵权
- Backer 沟通失败

## 2. 热门项目类

用于提炼成功项目的品类、卖点、定价、视频结构和传播点。

记录格式：

| 来源 | 项目名 | 平台 | 品类 | 筹款表现 | 核心卖点 | 视频/页面结构 | 可复用启发 |
|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |

常见标签：

- 户外
- 小工具
- 电子 DIY
- 宠物
- 桌面效率
- 智能硬件
- 漫画/文创
- 游戏周边

## 3. 项目分享类

用于沉淀不同品类的评估模板。

记录格式：

| 品类 | 用户痛点 | 适合众筹的原因 | 常见风险 | 必备素材 | 适合平台 |
|---|---|---|---|---|---|
|  |  |  |  |  |  |

## 4. 术语类

用于客户教育和报告解释。

| 术语 | 解释 | 在报告中的用法 |
|---|---|---|
| Kickstarter | 海外主流创意众筹平台 | 用于评估项目是否适合创意/硬件众筹 |
| Indiegogo | 海外主流众筹平台 | 适合部分硬件和延续销售项目 |
| Backer | 支持者/下单用户 | 报告中用于说明众筹用户沟通 |
| Pledge | 支持档位/认购金额 | 用于说明档位设计 |
| Early Bird | 早鸟档 | 用于首批冲量和制造稀缺感 |
| Stretch Goal | 解锁目标 | 用于拉长期提升客单价和传播 |
| Add-on | 加购项 | 用于提高客单价 |
| Pre-launch | 上线前预热 | 用于邮件池、广告测试和KOL蓄水 |

## 5. 出圈/整活类

用于补充传播创意模块，帮助项目从“能上线”提升到“有传播点”。

记录格式：

| 来源 | 创意类型 | 使用场景 | 适合品类 | 风险 | 可转成的执行动作 |
|---|---|---|---|---|---|
|  |  |  |  |  |  |

可关注方向：

- 视频前 3 秒钩子
- 反差演示
- 极端场景测试
- 创始人故事
- 用户挑战
- KOL 共创
- 评论区互动
- 限时解锁福利
